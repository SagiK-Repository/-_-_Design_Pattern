# Design Pattern

- 디자인 패턴
  - 디자인 패턴은 소프트웨어 개발에서 반복적으로 발생하는 문제를 효율적으로 해결하기 위한 재사용 가능한 해결책을 의미한다.
  - 코드의 재사용성, 모듈성, 유지 보수성을 높이는데 도움이 된다.
- 대표적인 패턴으로는 다음 3가지가 있다.
  - 생성 패턴 : 객체가 생성되는 방식을 중점
  - 구조 패턴 : 클래스와 객체를 더 큰 구조로 구성
  - 행위 패턴 : 객체 간의 상호작용과 책임 분배에 초점
